# Windows, Sheets & Dialogs

Windows contain the app interface. Sheets are modal dialogs anchored to a specific window, and panels are modeless secondary windows.

## Official Apple HIG Guidelines & Resources

- [Windows](https://developer.apple.com/design/human-interface-guidelines/windows)
- [Dialogs](https://developer.apple.com/design/human-interface-guidelines/dialogs)
- [Sheets](https://developer.apple.com/design/human-interface-guidelines/sheets)
- [Panels](https://developer.apple.com/design/human-interface-guidelines/panels)
- [Modality](https://developer.apple.com/design/human-interface-guidelines/modality)


## Key Design Rules & Constraints

- Use sheets for modal tasks targeted to a single window (e.g., save dialogs, settings overlays).
- Ensure windows have consistent window controls (close, minimize, zoom) and support resizing.
- Use panels for modeless tasks that need to remain visible while working in the main window (e.g., inspector, formatting panels).
- Provide clear, standard layouts for save, export, and alert dialogs.
